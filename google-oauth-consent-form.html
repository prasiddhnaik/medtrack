MedTrack Google OAuth Consent Form Values

Use these values in Google Cloud Console for the MedTrack OAuth consent screen. The app is currently local/testing, so public domain fields should be filled only after deployment.

App Information

App name MedTrack
User support email legoprasiddh@gmail.com
App logo Upload public/brand/medtrack-google-consent-logo.png. It is a PNG, 120px by 120px, and under 1 MB.

App Domain

Application home page For local testing, leave blank if Google allows it. After deployment, use your production URL, for example https://your-medtrack-domain.com/.
Application privacy policy link For local testing, leave blank if Google allows it. After deployment, use https://your-medtrack-domain.com/privacy.
Application Terms of Service link For local testing, leave blank if Google allows it. After deployment, use https://your-medtrack-domain.com/terms.
Do not enter localhost in App domain fields. Google OAuth redirect URIs can use localhost for development, but authorised domains on the consent screen should be real public domains you control.

Authorised Domains

Authorised domains For local testing, leave empty unless Google requires a domain. After deployment, add only the root production domain, for example your-medtrack-domain.com. Do not include https:// or paths.
